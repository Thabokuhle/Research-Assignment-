### 1. Main Types of Databases
- Relational Databases (SQL)
- NoSQL Databases

### 2. Relational Database Management System (RDBMS)
An RDBMS stores data in structured tables using rows and columns and allows data manipulation using SQL.

### 3. Primary Key & Foreign Key
- Primary Key: Uniquely identifies each record.
- Foreign Key: Links one table to another.

### 4. Database Normalization
The process of organizing data to reduce redundancy and improve data integrity.

### 5. Database Schema
A blueprint that defines tables, columns, relationships, and constraints.

### 6. Data Types
- Structured
- Semi-Structured
- Unstructured

### 7. Fact Table vs Dimension Table
- Fact Table: Numerical business metrics.
- Dimension Table: Descriptive attributes.

### 8. Data Model
Defines how data is structured and related within a database.

### 9. Database vs Data Warehouse vs Data Lake
- Database: Operational data
- Data Warehouse: Historical analytics data
- Data Lake: Raw structured & unstructured data

### 10. Data Mart
A subject-specific subset of a data warehouse.

### 11. SQL Query Language

Ensure data reliability and consistency.

### 14. Database Engine
Handles data storage, indexing, and query execution.

### 15. Views, Stored Procedures & Triggers
- View: Virtual table
- Stored Procedure: Reusable SQL code
- Trigger: Automatically executes on events

### 16. ETL vs ELT
- ETL: Transform before loading
- ELT: Transform after loading (cloud-based)

### 17. Batch vs Stream Processing
- Batch: Periodic processing
- Stream: Real-time processing

### 18. SQL Joins

### 19. Referential Integrity
Ensures relationships between tables remain valid.
